# Microsoft Entra ID

## Overview

This section documents the implementation of Microsoft Entra ID (formerly Azure Active Directory) to establish centralized identity and access management for the Microsoft 365 lab environment.

The implementation includes creating cloud users, assigning Microsoft 365 licenses, organizing users into security groups, and configuring Role-Based Access Control (RBAC) using Microsoft Entra built-in administrator roles.

## Objectives

- Configure Microsoft Entra ID users
- Assign Microsoft 365 E5 licenses
- Create Security Groups
- Configure Role-Based Access Control (RBAC)
- Demonstrate Microsoft Entra Privileged Identity Management (PIM)
- Validate identity configuration

## Technologies

- Microsoft Entra Admin Center

- Microsoft Entra ID

## Implementation

The following tasks were completed:

- Created cloud-only users
